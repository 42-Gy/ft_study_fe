# 학습 기록

## 한 줄 요약

-`MockApi`를 연결해서 데이터를 화면에 보여주고, 탭/필터링 등으로 화면이 바뀌는 마이페이지 대시보드 구현

## 구현 체크리스트

- [o] 내 정보 렌더링 (이름, 이메일, 수박씨 등)
- [o] 대여 현황 렌더링 (사물함 번호, 연체/대여 배지)
- [o] 아이템 기록 렌더링 (사용 중인건 초록색으로 표시)
- [o] 코인 기록 렌더링 (얻은 건 초록색, 쓴 건 빨간색)
- [o] 로딩/에러 상태 처리 
- [o] 탭 전환 구현 
- [o] 필터/정렬 구현 (날짜 최신순 정렬, 조건에 맞는 것만 골라보는 필터 적용)

## 어려웠던 점 / 해결

1. 에러 테스트 위해 error라 입력했으나, 에러 메시지 대신 정상 데이터가 조회됨
    - 해결 : mockApi.js의 검증 로직을 `if (token === "Bearer error")`로 수정함
2. css 클래스와 js 로직의 연결
    - good이란 문자열만 클래스로 추가했는데, 화면에 초록색 배지로 변하는 원리가 이해 안됐음
    - js는 상태만 변경하고, 실제 스타일은 css 파일에 정의되어 있는 것을 이해함
3. 날짜순 정렬(sort) 했더니 나중에 원래 순서대로 돌리려고 해도 원본 데이터가 이미 뒤섞여짐
    - 해결 : sort는 원본을 바꾸므로 `[...]`을 써서 원본(state)이 안전하게 보존되게 함
4. 데이터 없을 시 표 깨짐
    - 해결 : 내용물 비더라도 이름, 날짜 자리 등을 알려주는 헤더 추가함
5. `async/await` 역할 
    - `startWith` 써서 입력한 토큰 앞에 'Bearer '가 없으면 자동으로 붙여주고, 맞으면 그대로 쓰는 조건문 만들어서 토큰 처리 해결함
6. 클릭 시 목록이 툭 떨어지는 메뉴 어떻게 만드는가
    - 해결 : `<select>` `<option>` 태그 써서 선택할 수 있게 만듦
